This Python module is designed to help automate the extraction of WB projects sector codes and theme codes data.¶
The following is a brief demo explaining the some functionalities and possible use cases of the module.¶
Note: To use the module, access to IEG N-Drive is required. Additionally, for smooth functionality, it is advised that the directory in which this module is located be made the current working directory while using the module.¶

SECTOR CODES¶

To extract sector codes, user must first create a Sectors object and load this object with the WB project sectors data.¶
In [ ]:
#Import the Sectors object
from proj_codes import Sectors

#Create a Sectors object. 
sector_demo = Sectors()     #Users can give the object any name; "sector_demo" is used only for illustration.

Sectors object created.

The output has 1179 rows of data even though there are only 1149 unique projects. This is because some of those 1149 projects are mapped to both sectors codes, and thus are returned twice, each row indicating unique project ID-sector code combination.

Note that these 1149 projects are the projects where one or more of the sector codes provided account for at least 1% of the project. User can adjust this sector percentage threshold by setting the optional min_pct argument.

User can also customize the data extract by setting several other additional arguments. Example is shown below, but full usage information is available in the documentation for the get_project command.

In [ ]:
sector_demo.get_projects(sector_codes=my_codes,
                        min_pct=80,                 #Only return projects where any code in sector_codes account for 80% or more
                        start_FY=2000,              #Only return projects approved after FY2000
                        stop_FY=2010,               #Only return projects approved before FY2010
                        product_type=['L'],         #Only return lending projects
                        project_status=['closed'],  #Only return closed projects
                        include_AF=False)           #Do not include Additional Financing projects in the output
